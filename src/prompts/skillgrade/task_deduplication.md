## Task 相同的定义
- 测试相同的功能点/场景
- 触发相同的处理流程
- 考察相同的能力
- 仅在具体数值、名称等细节上有差异

## Task 不同的定义
- 测试不同功能点
- 触发不同处理流程
- 覆盖不同边界条件
- 考察不同能力维度

## 示例
- "转账100元给张三" vs "转账200元给李四"
  → same_task: true（都是测试完整转账，只是收款人和金额不同）

- "转账100元" vs "转账100元给张三"
  → same_task: false（前者测试信息缺失场景，后者测试完整场景）

- "转账500万" vs "转账100元"
  → same_task: false（前者测试大额边界，后者测试普通场景）

- "批量转账给10个人" vs "给张三转账"
  → same_task: false（前者测试批量处理，后者测试单个转账）

## 输出格式 (仅JSON)
```json
{{
  "same_task": true或false,
  "reasoning": "简要说明为什么是相同/不同的Task"
}}
```

只输出JSON，不要其他内容。
